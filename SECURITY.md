## Reporting a Vulnerability

At **Noventra Web Solutions (Pvt) Ltd**, we take the security of our software and our clients' data very seriously. If you find a security vulnerability, we appreciate your help in disclosing it to us in a responsible manner.

### How to Report
Please **do not** open a public GitHub issue for security vulnerabilities. Instead, send a detailed report to our security team:

* **Email:** info@noventra.com (or your preferred security email)
* **Subject:** Security Vulnerability Report - [Project Name]

### What to Include
To help us triage and fix the issue quickly, please include:
* A description of the vulnerability.
* Steps to reproduce the issue (PoC scripts or screenshots are helpful).
* The potential impact of the vulnerability.

### Our Response Process
1.  **Acknowledgment:** We will acknowledge receipt of your report within **48 hours**.
2.  **Investigation:** Our team will investigate the issue and may contact you for further details.
3.  **Resolution:** If the vulnerability is confirmed, we will work on a fix and notify you once it is deployed.
4.  **Disclosure:** We follow a responsible disclosure policy. We ask that you give us a reasonable amount of time to fix the issue before making any information public.
